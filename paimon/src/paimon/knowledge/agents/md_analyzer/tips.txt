<reading_tabular_data>
Thermodynamic data and analysis results are stored in HDF5 (.h5) format.
Before performing any analysis, first inspect the data structure and metadata using the inspect_h5 tool.
Whenever an .h5 file is provided, this inspection step must be performed prior to any data extraction or analysis.

After understanding the structure, read the required datasets along with their associated units as shown below.

Read specific data with units:
```python
import h5py
import numpy as np

with h5py.File("thermodynamic_data.h5", "r") as f:
    run = f["runs/run_0"]

    # Read datasets
    step = run["step"][:]
    temp = run["temperature"][:]

    # Get units and metadata from dataset attributes
    temp_units = run["temperature"].attrs["units"]
    temp_desc = run["temperature"].attrs.get("description", "")

    print(f"Temperature: {temp_desc}")
    print(f"Units: {temp_units}")
    print(f"Range: {temp.min():.1f} - {temp.max():.1f} {temp_units}")
```

Always check units before using numerical values. The .h5 file contains all necessary metadata.
</reading_tabular_data>

<trajectory_parsing>
IMPORTANT: Do not create MDAnalysis universe directly. Utilize get_mda_universe function instead.

```python
import h5py
from pathlib import Path
import MDAnalysis as mda
from simulation_util.io.universe import get_mda_universe

h5_file = "../01_MDSimulation/thermodynamic_data.h5"
h5_dir = Path(h5_file).parent

with h5py.File(h5_file, "r") as f:
    run = f["runs/run_0"]
    topology_file = run.attrs["topology_file"]
    trajectory_file = run.attrs["trajectory_file"]

u = get_mda_universe(
    topology=h5_dir / topology_file,
    trajectory=h5_dir / trajectory_file
)

for ts in u.trajectory:
    time.append(ts.time)  # The time unit in MDAnalysis is "ps"
    ...
```
</trajectory_parsing>

<ase_conversion>
Use ASE to write structure files in .extxyz format:
```python
from ase.io import write
from simulation_util.io.ase import mda_to_atoms

# Convert MDAnalysis to ASE Atoms object
# 'timestep' can be either an index or a Timestep object from u.trajectory
atoms = mda_to_atoms(u=u, timestep=100, pbc=True)
print(f"Atoms to save: {atoms}")  # Verify conversion
write("frame_100.extxyz", atoms)
</ase_conversion>

<important_rules>
- Before writing freud code, use the query_python_library tool to ask about the analysis you're interested in. This tool is for freud only.
- Use freud. It is fast, simple, and accurate. Typical analysis algorithms (RDF, MSD, ...) are already available from freud.
- Do not write a single Python script for multiple steps at once. Return the results of each step, inspect them, then write the next script.
- Don't accept results at face value. If something is orders of magnitude outside the expected range, it likely means (a) a bug in your code or (b) an unphysical setup. Track down the cause.
- Code-to-Physics/Math Transparency: When reporting a computed value, always show a minimal code snippet or echoed expression that makes the arithmetic explicit within the 'message_to_user' of the complete_task tool.
- Always verify units from .h5 file metadata before performing calculations or comparisons.
- If metadata is missing or unclear, inspect files first before proceeding with analysis.
</important_rules>
